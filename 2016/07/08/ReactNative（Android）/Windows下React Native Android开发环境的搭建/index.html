Windows下React Native Android开发环境的搭建

7月 08, 2016 发布在 ReactNative

随着React Native 的崛起，前段时间也大致瞟了一眼React Native，没做深入了解。
年初也有同学建议我关注React Native，毕竟现在纯原生开发有些落寞，再说这半年由于项目需要，

我们告别了纯粹的原生开发，直接使用cordova进行开发app。
当初刚开始写插件写的懵逼，不过github好多现成的插件可参考。
又学习了点AngularJS的知识。但是越发怀念原生开发。O(∩_∩)O~。

闲言碎语不多讲，昨天晚上真正着手React Native的学习。工欲善其事必先利其器。
搭建环境是第一步。开了文档，发现基本上全部都是Mac环境下的配置过程，毕竟那个什么。
自己也是根据React Native 中文网的文档，进行的配置。可以参考 大神
东方耀 手把手教React Native实战之环境搭建。

下面总结下自己的安装过程：

1.安装JDK
2.安装AndroidSDK
3.安装C++

（一般都有Microsoft Visual C++ 2013 Redistributable。。。之类的软件就够了）
对于安卓开发人员，以上步骤可以略过，这些都是必备的。但是有一点需要明确的就是，环境变量的配置。JAVA_HOME ANDROID_HOME ，要注意，这些做到命令行全局可以调用。其他的不用赘述

4.至于Git 的安装。不清楚哪里使用到了，感觉没什么必要。
5. Python的安装 从官网下载并安装python 2.7.x（3.x版本不行）
6.note.js 安装 （可以命令行安装，也可以直接下载软件）
7.安装 React Native 命令行工具。

我是采用直接github上的 react-native-master给down下来，解压然后定位到react-native-master文件夹下，react-native-cli目录，（shift+ctrl+右键）直接在此处打开命令窗口，输入 npm install -g 如图：

或者 npm install -g react-native-cli 也可以，但我没有试。

OK，大功告成，可以创建工程了。工程目录创建在哪？自己随意。然后
1.react-native init MyProject 初始化 创建工程（自己想要创建的工程名字） 下面是漫长的等待…主要是在创建node_modules，这个耗时。


出现这个就是成功了，看下现在工程的目录结构：

2.react-native start 启动服务 下面才算完成：

此时可以用浏览器访问http://localhost:8081/index.android.bundle?platform=android看看是否可以看到打包后的脚本（看到很长的js代码就对了）。
3.react-native run-android 运行 （首先要连接模拟器，或者真机）出现这个多半是环境的配置问题。

如果是安装成功，还报上面红色， adb devices 看看有连接的设备没有。
另外运行时候不要把刚才start的服务给关了。

真机上测试注意一点，要跟电脑在同一局域网内，不要开什么4g。如果项目运行后报红色警告，那就需要配置ip。
摇晃设备或按Menu键（Bluestacks模拟器按键盘上的菜单键，通常在右Ctrl的左边 或者左Windows键旁边），可以打开调试菜单，点击Dev Settings，

选Debug server host for device，输入你的正在运行packager的那台电脑的局域网IP加:8081，例如：192.168.0.38：8081（同时要保证手机和电脑在同一网段，且没有防火墙阻拦），

再按back键返回，再按Menu键，在调试菜单中选择Reload JS，就应该可以看到运行的结果了。

如果真实设备白屏但没有弹出任何报错，可以在安全中心里看看是不是应用的“悬浮窗”的权限被禁止了。

另外，习惯android studio开发的童鞋，如果做前端开发，可以使用webStorm（有破解版的），是studio的亲弟弟，写js、 h5这个没商量。
